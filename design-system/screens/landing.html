Under the hood

Agents that improve their own workflows

Most automation breaks in production because the harness — the prompts, tools, and orchestration around the model — is hand-built once and never evolves. intelligent-machines closes that loop automatically.

What is a harness?

The model is the engine. The harness is the whole vehicle — the system prompt, tools, memory, orchestration, evaluation logic, and retry behaviour that make the model actually useful in your specific workflow. Most agents look impressive in demos and break in production because their harness was engineered by hand and never updated.

The improvement loop
Worker Agent
Executes the task using the current harness — tools, prompts, and orchestration
Tries the task
result
Evaluator Agent
Reviews what happened, scores the result, and identifies exactly what went wrong
Finds failures
critique
Evolution Agent
Updates the harness — prompts, tool usage, orchestration — based on what failed
Improves harness
Worker tries again with an improved harness
What the loop can evolve
System prompts and task framing
Tool selection and usage patterns
Orchestration and step sequencing
Observation structure and memory
Retry logic and error handling
The bigger idea

The loop doesn't just improve one workflow. Over time, intelligent-machines learns which evolution strategies work fastest — and applies them when you deploy a new agent into a new domain.

You define the task. You define what success looks like. The system handles the rest.

"The best agent may not be the one with the best base model. It may be the one that can improve its own harness fastest."
